\section{Kiến trúc hệ thống}
\label{sec:system_architecture}

Hệ thống được thiết kế dựa trên mô hình Client-Server (Tham khảo Hình~\ref{fig:tech_architecture_overview}), phân tách rõ ràng giữa phần giao diện người dùng (Frontend) và phần xử lý nghiệp vụ, quản lý dữ liệu (Backend), đảm bảo tính mô-đun hóa, dễ dàng bảo trì và mở rộng.
\begin{itemize}[leftmargin=1.5em]
    \item \textbf{Client (Máy khách):} Đóng vai trò là Tầng trình bày (Presentation Layer), là một ứng dụng đơn trang (Single Page Application - SPA) chạy trên trình duyệt web của người dùng. Client chịu trách nhiệm hiển thị giao diện, xử lý tương tác người dùng và gọi API để giao tiếp với máy chủ.
    \item \textbf{Server (Máy chủ):} Chịu trách nhiệm xử lý các logic nghiệp vụ, xác thực quyền truy cập, tương tác với cơ sở dữ liệu và giao tiếp với các dịch vụ của bên thứ ba.
\end{itemize}

\section{Các dịch vụ tích hợp}
\label{sec:integration_services}

Để tối ưu hóa thời gian phát triển và phân tải cho máy chủ chính, hệ thống FitConnect tiến hành tích hợp các dịch vụ và API key từ bên thứ ba. Điểm nổi bật là toàn bộ các dịch vụ tích hợp được lựa chọn đều là gói miễn phí, phù hợp với quy mô dự án nhưng vẫn đảm bảo tính ổn định cao. Bao gồm:

\begin{itemize}[leftmargin=1.5em]
    \item \textbf{Cloudinary (API v2)~\cite{CloudinaryDocs}:} Nền tảng đám mây (CDN) chuyên dụng cho lưu trữ, tối ưu hóa và phân phối phương tiện truyền thông (hình ảnh, video). Hệ thống tích hợp Cloudinary bằng gói miễn phí để lưu hình ảnh, video, file,...
    
    \item \textbf{Groq API (Mô hình LLaMA 3)~\cite{GroqDocs}:} Cung cấp công cụ xử lý ngôn ngữ tự nhiên với tốc độ suy luận cực nhanh. Được tích hợp để hỗ trợ kiểm duyệt và tạo nội dung cộng đồng. Hệ thống đang sử dụng gói miễn phí cung cấp cho nhà phát triển.
    
    \item \textbf{Goong Maps API (v1)~\cite{GoongMapsAPI}:} Nền tảng bản đồ số cung cấp khả năng định vị, Geocoding và vẽ bản đồ. Tích hợp để theo dõi lộ trình chạy bộ/đạp xe ngoài trời thông qua GPS trên web. Hệ thống sử dụng gói miễn phí.
    
    \item \textbf{Google OAuth (v2.0)~\cite{GoogleOAuth}:} Cổng xác thực ủy quyền của Google (Google Identity Services) dùng cho tính năng đăng nhập một chạm (Single Sign-On). Đảm bảo bảo mật tài khoản người dùng theo tiêu chuẩn OAuth 2.0 toàn cầu và được cung cấp hoàn toàn miễn phí.
    
    \item \textbf{Gmail SMTP (TLS/SSL)~\cite{NodemailerDocs}:} Giao thức gửi thư điện tử được tích hợp thông qua thư viện Nodemailer kết hợp với Mật khẩu ứng dụng (App Passwords) của Google. Dùng để gửi email chứa mã xác thực (OTP) hoặc cấp lại mật khẩu một cách nhanh chóng, miễn phí mà không cần thuê máy chủ Mail Server chuyên dụng.
\end{itemize}

\section{Công nghệ sử dụng}
\label{sec:tech_stack}

Để hiện thực hóa kiến trúc trên, dự án sử dụng hệ sinh thái JavaScript/TypeScript toàn diện nhằm tạo sự đồng nhất từ Client đến Server. Dưới đây là các công nghệ cốt lõi được sử dụng trong hệ thống cùng với phiên bản và mục đích cụ thể:

\subsubsection*{1. Công nghệ Frontend (Giao diện người dùng)}
\begin{itemize}[leftmargin=1.5em]
    \item \textbf{React (v18.2)~\cite{ReactJSDocs}:} Thư viện UI cốt lõi để xây dựng giao diện theo mô hình Component, giúp dễ dàng tái sử dụng mã nguồn và quản lý trạng thái hiển thị.
    \item \textbf{Vite (v4.4)~\cite{ViteDocs}:} Công cụ build frontend thế hệ mới (thay thế Webpack) giúp tối ưu hóa tốc độ khởi động server lúc phát triển và tối ưu file tĩnh khi triển khai lên production.
    \item \textbf{Tailwind CSS (v3.3)~\cite{TailwindCSSDocs}:} Framework CSS tiện ích (Utility-first CSS) hỗ trợ tạo kiểu trực tiếp trên các thẻ HTML, giúp dựng giao diện linh hoạt (Responsive) nhanh chóng.
    \item \textbf{TanStack Query / React Query (v5.28)~\cite{TanStackQuery}:} Thư viện quản lý trạng thái máy chủ (Server-state management), tự động xử lý caching, đồng bộ dữ liệu nền và tối ưu số lượng gọi API.
    \item \textbf{React Router (v6.16)~\cite{ReactRouterDocs}:} Quản lý điều hướng (Routing) chuyển đổi giữa các màn hình trong ứng dụng Single Page Application mà không cần tải lại toàn bộ trang trình duyệt.
\end{itemize}

\subsubsection*{2. Công nghệ Backend (Máy chủ API)}
\begin{itemize}[leftmargin=1.5em]
    \item \textbf{Node.js (v20.9 LTS)~\cite{NodeJSDocs}:} Môi trường thực thi mã JavaScript đa nền tảng. Chạy trên kiến trúc hướng sự kiện và Non-blocking I/O, cực kỳ phù hợp cho các API yêu cầu tốc độ đáp ứng cao và kết nối thời gian thực.
    \item \textbf{Express.js (v4.18)~\cite{ExpressJSDocs}:} Web Framework siêu nhẹ của Node.js, cung cấp hệ thống Routing mạnh mẽ và cơ chế Middleware linh hoạt để thiết kế các RESTful API.
    \item \textbf{TypeScript (v5.2)~\cite{TypeScriptDocs}:} Ngôn ngữ lập trình phiên bản chặt chẽ hơn của JavaScript (Typed Superset). Giúp bắt lỗi kiểu dữ liệu (Static Typing) từ lúc viết code, tăng tính tự động nhắc lệnh (IntelliSense) và độ tin cậy cho toàn dự án.
    \item \textbf{Socket.IO (v4.7)~\cite{SocketIODocs}:} Thư viện hỗ trợ kết nối và giao tiếp hai chiều thời gian thực (Real-time) dựa trên WebSockets. Được ứng dụng để đẩy hệ thống thông báo cho người dùng.
    \item \textbf{JSON Web Token - JWT (v9.0)~\cite{JWTSpec}:} Chuẩn mã hóa chuỗi thông tin để xác thực (Authentication). Cho phép kiểm tra danh tính người dùng theo phương thức Stateless (không lưu phiên trên RAM máy chủ) qua mỗi request.
\end{itemize}

\section{Thiết kế cơ sở dữ liệu}
\label{sec:database_design}

Hệ thống sử dụng \textbf{MongoDB}~\cite{MongoDBDocs} --- cơ sở dữ liệu NoSQL hướng tài liệu. Dữ liệu được tổ chức thành các \textbf{collection}, mỗi collection chứa nhiều \textbf{document} dạng JSON/BSON linh hoạt. Quan hệ giữa các collection thực hiện qua \textbf{referencing} (lưu \texttt{ObjectId}) hoặc \textbf{embedding} (nhúng sub-document trực tiếp). Hệ thống FitConnect bao gồm 13 collection chính:

\section{Kết luận chương}
\label{sec:tech_conclusion}

Chương 4 đã trình bày chi tiết về nền tảng kỹ thuật và thiết kế hệ thống của dự án FitConnect. Thông qua việc phân tích kiến trúc Client-Server, lựa chọn hệ sinh thái công nghệ hiện đại (MERN Stack kết hợp TypeScript) và thiết kế cơ sở dữ liệu NoSQL linh hoạt với 13 collection được chuẩn hóa, hệ thống đã hình thành một khung sườn vững chắc, đáp ứng tốt các yêu cầu về hiệu năng và khả năng mở rộng. 

Điểm nhấn của thiết kế nằm ở việc khai thác sức mạnh từ các dịch vụ đám mây và API tích hợp (Cloudinary, Goong Maps, Groq AI), giúp tối ưu hóa tài nguyên server trong khi vẫn đảm bảo cung cấp các tính năng thông minh và đa dạng cho người dùng. Những thiết kế chi tiết về cấu trúc dữ liệu và kiến trúc logic này chính là cơ sở quan trọng để tiến hành hiện thực hóa hệ thống thực tế, nội dung này sẽ được trình bày chi tiết trong Chương 5.
